## 前端开发中使用 Mock Data 的场景

前端开发中使用 Mock Data 的场景很广泛，主要在后端接口没完成或者不可以用的时候：
- 场景：后端API尚未实现，前端需要先行开发界面和交互逻辑。
- 场景：需要快速构建产品原型向客户或产品经理演示。
- 单元测试阶段，场景：测试组件或函数在特定数据下的行为和渲染结果。
   - 场景：测试多个组件或模块协同工作时的表现。
   - 场景：模拟真实用户操作流程，从界面到数据层的完整测试。
 
- 特定的功能场景：
   - 场景：模拟各种异常情况，测试应用的健壮性。
   - 场景：测试分页组件、无限滚动或懒加载功能。
   - 场景：测试搜索框、筛选器等功能。

最佳实践建议
- 保持Mock数据与API契约一致：确保Mock数据结构与后端API文档定义的响应结构一致。
- 使用TypeScript/类型定义：为Mock数据定义类型，提高代码可靠性。
- 集中管理Mock数据：将Mock数据放在统一的目录结构中，便于维护。
- 区分环境：只在开发和测试环境使用Mock，生产环境始终使用真实API。
- 定期更新：随着API变更，及时更新Mock数据。
- 模拟真实场景：包括成功响应、错误响应、边缘情况等。
- 文档化：为Mock数据的用途和使用方式添加注释。

通过合理使用Mock Data，前端团队可以大幅提高开发效率，减少对后端进度的依赖，同时提升代码质量和应用稳定性。

Mock工具库
- MSW (Mock Service Worker)：浏览器中拦截请求
- json-server：快速创建REST API mock服务器
- Faker.js：生成逼真的模拟数据
- Mirage JS：完整的API mocking框架
